# 모임온(MoimON) — 클로드디자인 화면 브리핑 모음

클로드디자인(claude.ai/design)에 붙여넣어 화면을 디자인할 때 쓰는 브리핑입니다.
**매 화면마다 맨 앞에 [디자인 시스템] 블록을 먼저 붙이고**, 그 아래에 해당 화면 브리핑을 이어 붙이세요.
완성되면 핸드오프 번들 링크(api.anthropic.com/v1/design/...)를 개발자(클로드)에게 주면 그대로 이식됩니다.

추천 순서: ① 앱 셸 → ② 대시보드 → ③ 회원 목록 → 출석·식대 / 회계 보고서 → 공지·콘텐츠.
한 번에 다 하지 말고 한두 화면씩 만들어 넘기며 검증하세요.

## [공통] 디자인 시스템 블록 — 매 화면 맨 앞에 붙이기

```
[디자인 시스템 — 모임온(MoimON), 랜딩과 통일]
- 톤: 깔끔·따뜻·신뢰. 토스/노션처럼 여백 넉넉, 한국형 SaaS, 모바일 우선
- 폰트: Pretendard. 아이콘: lucide
- 색: 포인트 #0066cc(진한 #0052a8, 연한 #e8f1fc, 더연한 #f3f8fe)
  글자 #16181d(제목)/#3d424d(본문)/#767d8a(보조), 경계선 #ecedf0
  배경 #ffffff, 따뜻한 회색 배경 #fafafb, 초록(참석/입금) #0a7d3f·#e4f6ec
- 모서리: 버튼 14px, 카드 20px / 그림자 옅게 / 둥근 버튼
- 상단: 좌측 "ON 모임온"(파란 ON 배지) 로고, 한국어 UI
- 표/리스트는 여백 충분히, 모바일 1열·데스크톱 다열
```

## ① 앱 셸 (공통 헤더 + 메뉴)

```
[화면] 로그인 후 공통 레이아웃(헤더 + 메뉴 바) — 모든 내부 화면 위에 깔림
[목적] 어디서나 같은 틀로 메뉴 이동 + 내가 누구인지(임원/회원) 표시
[구성]
1) 상단 헤더: 좌측 "ON 모임온" 로고, 우측에 내 이름/이메일 + 역할 배지(임원/회원) + 로그아웃
2) 그 아래 메뉴 바(가로 탭): 대시보드·회원관리·연간일정·출석식대·회계·콘텐츠·공지·출석통계
   - 현재 메뉴는 파란색 강조(밑줄 또는 알약 배경)
   - 회원(비임원)은 메뉴가 "공지"만 보임
3) 모바일: 메뉴가 가로 스크롤 또는 햄버거
깔끔한 흰 배경, 메뉴는 #3d424d 글자·hover 시 파란색.
```

## ② 대시보드 (임원용 첫 화면)

```
[화면] 임원용 대시보드 (로그인 후 첫 화면)
[목적] 이번 주 모임 현황을 한눈에. 자주 쓰는 메뉴로 빠르게 이동.
[구성]
1) 인사 한 줄 + 이번 주(또는 다음) 모임 카드: 날짜·요일, 온라인/오프라인 배지, 회차, 프로그램(예배/포럼 등), 주제·강사
2) 요약 통계 4칸 카드: 전체 회원 / 활동 회원 / 이번 달 수입·지출(원) / 다가오는 일정 수
3) 바로가기 그리드(아이콘+라벨): 회원관리·연간일정·출석식대·회계·콘텐츠생성·공지·출석통계
4) 최근 공지 3개 리스트(제목·카테고리·날짜)
한국어, 카드 기반. 모바일 1열, 데스크톱 통계 4열·바로가기 3~4열.
```

## ③ 회원 목록 (노션식 표)

```
[화면] 회원 목록 (임원용 메인)
[목적] 30~50명 회원을 표로 관리. 정렬·필터·검색·열 선택.
[구성]
1) 요약 통계 4칸: 전체 회원 / 활동 / 정회원 / 부부회원
2) 우측 상단 버튼: "엑셀 업로드", "+ 회원 추가"
3) 필터 줄: 등록(등록/비등록)·등급(명예·정·부부·준·신입회원)·상태(활동·유보·등록전·OB) + 검색창 + 열 선택
4) 노션식 표: 이름·성별·연락처·등급·상태·배우자·업종·회사·직책·비전스쿨·리더십스쿨·가입일·태그
   - 등급/상태는 색 배지(알약), 행 hover 강조, 이름 클릭 시 상세
모바일에선 카드형(이름+등급배지+연락처) 또는 가로 스크롤 표. 토스/노션 느낌의 깔끔한 표.
```

## ④ 회원 상세 · 편집

```
[화면] 회원 상세 보기 + 편집 폼
[목적] 한 회원의 모든 정보를 보고 수정.
[구성]
1) 상단: 이름(크게) + 등급·상태 배지 + 연락처, 우측 "편집/저장" 버튼
2) 정보 카드 그룹(섹션별): 기본(성별·연락처·배우자), 직업(업종·회사·직책),
   교육(비전스쿨·리더십스쿨), 차량(차종·차번호·주차등록), 가입일, 태그(이력 칩)
3) 편집 모드: 같은 레이아웃에 입력칸(셀렉트/텍스트), 하단 저장·취소
민감정보(주민번호·계좌)는 없음. 카드 기반, 라벨은 보조색·값은 진한색. 모바일 1열.
```

## ⑤ 연간 일정

```
[화면] 연간 일정 (금요일 정기모임 표)
[목적] 한 해 모든 금요일 모임의 회차·형태·프로그램·주제·강사를 관리.
[구성]
1) 상단: 연도, "자동 생성", "엑셀 업로드" 버튼, 기준 회차 설정
2) 표(행=날짜): 날짜(요일)·회차·형태(온라인/오프라인/휴회/미정 배지)·프로그램(예배·포럼·회만시·특강·기타행사 셀렉트)·주제·강사·비고
   - 온/오프/미정은 색 배지, 공휴일·5번째 금요일은 '미정' 회색
   - 지난 모임은 흐리게, 다가오는 모임은 강조
모바일 카드형. 토스 느낌의 가벼운 표.
```

## ⑥ 출석 · 식대 (임원 관리)

```
[화면] 출석·식대 관리 (임원)
[목적] 회차를 골라 출석 현황을 보고, QR 자가 체크인을 열고, 식대 안내를 설정.
[구성]
1) 회차 선택(드롭다운/탭): 날짜·회차·온오프
2) 온라인 회차면 "식대 없음" 안내, 오프라인이면 식대 관련 표시
3) QR 코드 카드: 회원이 폰으로 스캔해 자가 체크인 (큰 QR + 링크 복사)
4) 출석 명단: 회원별 출석/결석 토글, 출석 인원·결석 인원 통계
5) 식대 입금 설정 카드: 식대 금액 · 입금 계좌 · (선택)송금 링크
6) 우측 상단 "출석 통계" 이동 버튼
오프라인 회차에만 식대/입금 열 표시. 카드+표. 모바일 1열.
```

## ⑦ 출석 체크인 (회원용 · QR 진입)

```
[화면] QR 체크인 (회원이 폰으로 들어오는 화면, 로그인 없이)
[목적] 회원이 명단에서 본인을 찾아 탭 한 번으로 출석 + 식대 안내 확인.
[구성]
1) 상단: 모임 날짜·회차·형태, "본인을 찾아 체크인하세요" 안내
2) 회원 명단(이름 버튼 그리드/리스트): 탭하면 출석 처리, 이미 출석한 사람은 "참석" 표시 + "취소" 버튼
3) 오프라인이면 출석 직후 화면 가운데 모달: "식대 안내 — N원 · 계좌 복사" (계좌 복사 버튼)
   온라인이면 "식대 없음" 표시
모바일 전용(폰 세로). 큰 터치 버튼, 글자 큼. 공통 헤더/메뉴 없음(단독 화면).
```

## ⑧ 출석 통계

```
[화면] 출석 통계 (임원)
[목적] 회차별·프로그램별·회원별 출석률 + 송년회 출석상 후보.
[구성]
1) 연도 필터(알약 버튼), 상단 요약 3칸: 진행된 모임 / 평균 출석률 / 출석상 후보(90%+)
2) 프로그램별 출석률 표(예배·포럼·회만시·특강·기타행사): 모임수·평균출석·출석률(막대바)
3) 회차별 출석률 표: 회차·날짜·주제·출석인원·출석률 막대
4) 회원별 출석률 표: 순위·이름·출석/전체·출석률 막대 + 90%↑ "🏅 출석상" 배지
막대 그래프는 파란색, 깔끔한 표. 모바일 가로 스크롤.
```

## ⑨ 회계 — 거래 가져오기 (엑셀 업로드)

```
[화면] 거래 내역 가져오기 (은행 엑셀 업로드)
[목적] 통장 엑셀을 올리면 자동으로 입출금·식대/회계로 분류해 미리보기 후 저장.
[구성]
1) 업로드 영역(드래그&드롭 카드) + "엑셀 선택" 버튼
2) 분석 결과 미리보기 표: 날짜·적요(내용)·금액·입출금·자동분류(연회비·식대·강사료 등)·신뢰도
   - 자동분류 결과는 색 배지, 애매한 건 강조 + 수정 셀렉트
3) 하단: "N건 저장" 버튼, 분류 통계 요약
민감정보(계좌번호) 저장 안 함 — 이름·금액·내용만. 깔끔한 표.
```


## ⑪ 회계 보고서

```
[화면] 회계 보고서 (월별·분기별·연간)
[목적] 정산기간의 수입지출·연회비현황·식대정산·잔액을 보고서로. 인쇄/PDF.
[구성]
1) 기간 선택(월/분기/연/전체 탭) + 정산기간 표시(예: 2026.01.01~03.31)
2) 수입·지출 요약 카드 + 분류별 표(연회비·강사료·식대·구독 등)
3) 연회비 현황: 정회원·부부회원·준회원 납부 인원/총원
4) 식대 정산표: 회차별 식대 수입·지출·잔액
5) 잔액(이월 포함)
6) 우측 상단 "인쇄/PDF" 버튼 — 인쇄 시 상단 헤더·메뉴 숨기고 보고서만
공식 보고서 느낌으로 단정하게. 표 중심, 흑백 인쇄에도 또렷하게.
```

## ⑫ 콘텐츠 생성 (주간 포스터 + 안내글)

```
[화면] 콘텐츠 생성 — 주간 포스터 편집기 + 안내글
[목적] 이번 주 모임 포스터와 안내글을 몇 번 클릭으로 만들고 공지에 게시.
[구성]
1) 좌측 입력: 이번 주 제목·말씀구절·강사·일시 (연간일정에서 자동 채움)
2) 가운데 포스터 미리보기(세로 카드): 배경 + 글자(제목/카테고리/말씀/강사/일시) + CBMC 로고
3) 우측 탭 패널: 배경(보관함/스톡사진/AI생성/테마·단색), 글자(내용·크기·위치·폰트·색), 틀(템플릿)
4) 하단: "PNG 저장", "공지에 게시" 버튼 + 안내글 미리보기
편집 도구는 탭으로 단순하게. 포스터는 인스타 세로 비율. 데스크톱 3열(입력·미리보기·도구).
```

## ⑬ 경조사 안내장

```
[화면] 경조사 안내 만들기
[목적] 모바일 청첩장/부고 링크를 붙이거나 이미지를 올리면 내용 자동 추출 → 안내 카드 생성 → 공지 게시.

2) 자동 채워진 폼: 종류(결혼·부고·돌 등)·누구·일시·장소·링크·비고
3) 미리보기 카드(따뜻한 톤) + "공지에 게시" 버튼
간결한 폼 + 카드. 모바일 1열.
```

```
[화면] 공지 게시판
[목적] 임원이 공지·포스터·경조사를 올리고, 회원은 읽기.
```

## ⑮ 회원용 홈

```
[화면] 로그인 (매직링크)

[구성]
1) 가운데 카드: "ON 모임온" 로고 + "모임 운영을 ON" 한 줄

3) 안내문: "입력하신 메일로 링크를 보내드려요. 비밀번호 없이 안전하게."
4) 전송 후: "메일을 확인하세요" 상태 카드
랜딩과 같은 톤, 가운데 정렬, 흰 배경 + 옅은 파란 글로우. 모바일 우선.
```

## ⑰ 새서울 CBMC — 첫 화면 (환영 + 로그인) ★새서울 전용

> 판매용 '모임온' 광고가 아니라, **새서울 앱의 현관**이에요.
> 모임온 디자인 시스템(색·폰트·둥근버튼)은 그대로 쓰되 **브랜드는 '새서울 CBMC'**.

```
[화면] 새서울 CBMC — 첫 화면(환영 + 로그인)
[맥락] '새서울 CBMC 아름다운 만남' — 기독실업인회 새서울지회의 금요 조찬 정기모임 운영 앱.
로그인 안 한 방문자가 보는 첫 화면. 회원·임원이 여기서 로그인해 앱으로 들어감.
'모임온' 제품 광고가 아니라 '우리 모임(새서울 CBMC)'의 따뜻한 현관.
[톤] 모임온 디자인 시스템(색 #0066cc·Pretendard·둥근버튼)과 통일하되 브랜드는 '새서울 CBMC'.
따뜻·단정·신뢰. 흰 배경 + 옅은 파란 글로우. 모바일 우선, 한 페이지(짧게).
[구성]
1) 상단: 새서울 CBMC 로고(없으면 'CBMC' 파란 배지) + '새서울 CBMC' 텍스트
   ※ '모임온' 로고 자리에 새서울 정체성을 둠
2) 히어로(가운데 정렬):
   - 큰 환영 제목: "새서울 CBMC\n아름다운 만남" ('아름다운 만남'에 파란 포인트)
   - 한 줄 소개: "회원·출석·식대·회계·공지를 한 곳에서. 우리 모임 운영 공간이에요."
3) 로그인 카드(가운데 흰 카드):
   - 이메일 입력칸 + "로그인 링크 받기"(파란 큰 버튼)
   - 안내: "비밀번호 없이, 등록된 메일로 안전하게 로그인해요."
   - 전송 후 상태 카드: "메일을 확인하세요 — 로그인 링크를 보냈어요"
4) 작은 안내 두 줄:
   - "처음이신가요? 간사(임원)가 등록한 이메일로 로그인하세요."
   - "출석 체크는 모임 현장 QR로 따로 진행돼요."
5) 푸터: "새서울 CBMC · 아름다운 만남" + 아주 작게 "powered by 모임온"
[비고] 주민번호·계좌 등 민감정보 없음. 큰 터치 버튼·큰 글자(고령 회원 배려). 한국어.
```
