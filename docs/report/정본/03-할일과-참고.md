# 03 — 할일과 참고

> 추측 금지. 불명확한 것은 전부 여기 "확인 필요"로. 향후 방향·TODO도 여기.

## 확인 필요 (코드만으로 단정 못 한 것)
- **Supabase Redirect URLs**: 프로덕션 카카오 로그인이 실제 성공했으므로 `https://cbmc-app.vercel.app/auth/callback`은 등록된 것으로 보이나, 대시보드에서 최종 확인 권장(도메인 추가 시 갱신 필요).
- **`officer_terms`(연도별 임원) 사용처**: 테이블은 있으나 이를 직접 편집/표시하는 전용 화면(라우트)을 코드에서 확인하지 못함. 어디서 쓰는지(또는 미사용인지) 확인 필요.
- **`/dashboard`·`/content` 세부 동작**: 페이지 존재는 확인했으나 내부 위젯/생성 로직 세부는 본문에 단정하지 않음. 정밀 기재 필요 시 해당 컴포넌트 직접 확인.
- **회원 메뉴 범위**: 현재 회원(비임원)은 네비에 "공지" 한 개만 노출(`SiteNav` MEMBER). 회원 전용 추가 화면 계획이 있는지 확인 필요.
- **카카오 로그인 e2e(타 계정)**: 운영자 본인 외 실제 회원 계정으로의 로그인 검증은 아직 미확인.

## 향후 방향 / 미착수
- **카카오 사용자 ↔ 회원(members) 매칭**: 카카오로 들어온 사용자(이메일 없을 수 있음)를 기존 `members` 레코드와 연결하는 로직 없음. 식별·연결 방식 설계 필요.
- **역할 자동 승격**: 신규 로그인은 `guest` 고정. `guest → member` 승격 정책/화면 필요.
- **다지회 지원**: 지금은 `'새서울'` 하드코딩 + RLS가 chapter_id를 안 봄. 다지회로 가려면 (1) 앱 쿼리의 chapter_id 동적화, (2) RLS에 chapter_id 격리 추가가 동시에 필요.
- **공통 인증 게이트**: 페이지마다 흩어진 `getUser()` 검사를 미들웨어/레이아웃 레벨로 정리 검토.
