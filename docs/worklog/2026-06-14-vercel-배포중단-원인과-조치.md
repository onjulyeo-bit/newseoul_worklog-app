## 근본 원인
끊긴 지점은 **Vercel ↔ GitHub 연동**이다(코드/병합/로컬 git은 전부 정상).
구체적으로: **Vercel용 GitHub App의 "select repositories" 허용 목록에서 `newseoul_worklog-app`가 빠지면서**, Vercel이 이 저장소의 push 이벤트(webhook)를 못 받게 됐고, 그 결과 cbmc-app 프로젝트의 Git 연결도 끊어졌다 → 자동배포 중단.

언제 빠졌는지 정확한 트리거는 로그로 단정 불가(GitHub은 앱 설치 repo목록 변경 이력을 일반 API로 노출하지 않음). 다만 정황상 **맥스튜디오→맥프로 기기 이전 중 GitHub↔Vercel 권한을 재인증/재설정하는 과정에서 이 저장소가 다시 선택되지 않은 것**으로 보인다. (사용자가 "깃허브 저장 단계에서 연결이 끊겼다는 메시지"를 본 시점과 일치)

## 재발 방지 (조치)
1. **[권장·핵심] Vercel GitHub App을 "All repositories"로 변경**
   GitHub → Settings → Applications → Vercel → Configure → Repository access → **All repositories** → Save.
   → 새 저장소/재연결 시에도 자동으로 접근이 유지되어, "목록에서 빠져 조용히 배포 안 됨" 사고가 원천 차단됨.
   (트레이드오프: Vercel이 전체 저장소 접근. 개인 계정이면 일반적으로 허용 가능. 최소권한을 원하면 "select" 유지하되, 새 repo 만들 때마다 목록에 추가하는 걸 반드시 기억해야 함.)
2. **배포 확인 습관**: main에 push한 뒤에는 항상 Vercel Deployments 탭에서 새 빌드(해당 커밋 해시)가 뜨는지 1회 확인. 안 뜨면 = 연동 끊김 신호.
3. **기기 이전 시 체크리스트**: 새 맥에서 작업 시작 전 — ① `git remote -v` 가 `newseoul_worklog-app` 가리키는지 ② Vercel cbmc-app 프로젝트에 저장소 연결돼 있는지(카드에 GitHub 배지 표시) 확인.

## 빠른 진단 가이드 (다음에 "배포가 안 돼요" 할 때)
1. `git show origin/main:<파일>` 로 코드가 main에 있는지 → 있으면 코드 문제 아님
2. Vercel Deployments 탭에 해당 커밋 배포가 있는지 → 없으면 연동 문제
3. Vercel 프로젝트 카드/Settings→Git 에 저장소 연결됐는지 → "Connect" 보이면 재연결
4. GitHub → Applications → Vercel → Repository access 에 `newseoul_worklog-app` 있는지
