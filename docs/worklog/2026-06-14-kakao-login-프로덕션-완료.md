# 2026-06-14 — 카카오 로그인 프로덕션 완료 (KOE205 해결 포함)

카카오 로그인 버튼이 프로덕션(cbmc-app.vercel.app)에서 **실제 로그인까지 성공**. 회원용 카카오 로그인 라이브.

## 배포까지의 두 가지 막힘과 해결
1. **배포가 안 됨** → Vercel↔GitHub 연동 끊김이 원인. 저장소 재연결 + GitHub App을 All repositories로 변경. (상세: 2026-06-14-vercel-배포중단-원인과-조치.md)
2. **카카오 버튼 클릭 시 KOE205 "잘못된 요청"** → 아래로 해결.

## KOE205 원인 (중요)
- **Supabase는 카카오 로그인 시 `account_email` scope를 항상 강제 요청**한다. 실제 카카오로 가는 scope = `account_email profile_image profile_nickname`.
- 클라이언트 `signInWithOAuth`의 `options.scopes`는 **덧붙기만** 되고 account_email을 못 뺀다(테스트로 확인: `scopes:'profile_nickname'` 줘도 account_email 그대로 포함).
- 그런데 **개인 개발자 카카오 앱은 이메일이 "권한 없음"** → 미설정 동의항목 요청 → KOE205.
- 즉 코드 문제 아님. **카카오 앱에서 이메일 권한을 열어야** 함.

## 해결 절차 (카카오 개발자 콘솔, 코드/배포 변경 없음)
1. 앱 → 앱 설정 → 일반 → **앱 아이콘 등록**(비즈 앱 전환 선행조건)
2. 비즈니스 정보 → **개인 개발자 비즈 앱 전환** (사업자번호 불필요, 본인인증+카카오비즈니스 약관동의). 전환 목적="이메일 필수 동의" 선택 후 저장.
3. 카카오 로그인 → 동의항목 → **카카오계정(이메일) → "선택 동의"** 로 설정(+동의 목적 입력, 저장).
   - "선택 동의"로 한 이유: 우리 설계가 "이메일 없어도 로그인 통과"(Supabase Allow users without email ON). 선택 동의면 account_email이 "설정된 항목"이 되어 KOE205 사라지고, 이메일 미동의 사용자도 로그인 완료됨.
4. 결과: 닉네임·프로필사진·이메일 모두 선택 동의 → 프로덕션 카카오 로그인 성공 확인.
